# E-Commerce-Sales-Analysis

## Overview
I am excited to present this end‑to‑end E‑Commerce Data Analysis project, designed to tackle a complex dataset distributed across multiple relational CSV files. This dual‑technology solution integrates SQL for robust querying and Python for advanced Exploratory Data Analysis (EDA) and visualization.
By bridging the gap between raw, fragmented data and clear business intelligence, the project uncovers actionable insights into customer purchasing behavior, product performance, and revenue growth trends over time.
The repository includes all SQL queries, Python scripts, and presentation materials, providing a complete view of the data lifecycle—from ingestion and transformation to visualization and strategic insight generation.

## Key Features
* Customer Insights: Top cities by customer count, average products per order, and spending patterns.

* Product Analysis: Top categories by sales and revenue contribution, highlighting diversification opportunities.

* Seller Performance: Ranking of sellers by total revenue to identify critical partners.

* Revenue Trends: Monthly vs cumulative revenue, year‑over‑year growth analysis.

* Visualizations: Clear, annotated charts (bar, pie, line) created with Matplotlib and Seaborn.

## Tech Stack
* Languages: Python, SQL

* Libraries: Pandas, Matplotlib, Seaborn

* Tools: Jupyter Notebook, MySQL workbench

* Version Control: GitHub for portfolio showcase

## Business Insights
* High‑engagement cities show bulk buying behavior → logistics & inventory planning.
* A few product categories dominate revenue → risk of dependency, need for diversification.
* Top sellers drive majority of revenue → strategic partnerships critical.
* Seasonal peaks and YoY growth trends → guide marketing campaigns and launches.
